# PRODUCT PERFORMANCE KPI - FIXES & DYNAMIC SYNC CONFIRMATION

## Date: January 22, 2026

## ✅ FIXES COMPLETED

### 1. Fixed Data Source (Critical)
**Problem:** Queries were using non-existent view `vw_unified_sales_complete`
**Solution:** Changed to query `fp_actualcommon` directly with proper JOINs

### 2. Fixed Division Filtering
**Problem:** JOIN used `division_code` which excluded BF data (61.7% of sales!)
**Solution:** Changed JOIN to use `admin_division_code` to include both FP and BF divisions

### 3. Fixed Value Type Aggregation
**Problem:** Queries only aggregated AMOUNT, missing KGS and MORM
**Solution:** Added CROSS JOIN LATERAL to unpivot qty_kgs, amount, and morm columns

### 4. Synced Material/Process Mappings
**Problem:** `fp_product_group_unified` had NULL material/process for several products
**Solution:** 
- Synced data from `fp_material_percentages` (source of truth)
- Fixed 3 products: Labels, Others, POF Films Products
- Fixed Services Charges normalization

### 5. Excluded "Other" from KPIs
**Problem:** "Other" process/material showing in KPIs (miscellaneous items)
**Solution:** Added filter `LOWER(pg.process) != 'other'` and `LOWER(pg.material) != 'other'`

### 6. Implemented Dynamic Sync ⭐
**Problem:** Changes in Material Percentages page weren't automatically reflected in KPIs
**Solution:** Added automatic sync from `fp_material_percentages` → `fp_product_group_unified`

## 📊 QUERY ARCHITECTURE

### Tables Used:
1. **fp_actualcommon** - Main actual sales data (50,780 rows)
   - Contains: qty_kgs, amount, morm, pgcombine
   - Filter by: `admin_division_code = 'FP'` (includes FP + BF)

2. **fp_product_group_unified** - Master product data
   - Contains: material, process, normalized_name
   - Synced from: fp_material_percentages

3. **fp_material_percentages** - SOURCE OF TRUTH
   - Material composition (PE%, BOPP%, PET%, etc.)
   - Material category (PE, Non PE, Other)
   - Process category (Printed, Plain, Other)

### JOIN Logic:
```sql
FROM fp_actualcommon d
LEFT JOIN fp_product_group_unified pg 
  ON UPPER(TRIM(d.pgcombine)) = pg.normalized_name
  AND pg.division = d.admin_division_code  -- ✅ Critical!
```
